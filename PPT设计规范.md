# HTML PPT 设计规范

> 记录所有经过反复调整确认的设计细节，以后做新 PPT 时直接对照执行，减少来回修改。
> 最后更新：2026-07-17

---

## 一、整体布局

### 1.1 内容垂直居中
- **所有页面内容必须垂直居中**，不能底部留大片空白
- 实现方式：`.frame { justify-content: center }` 或 `.content-center` 工具类
- 内容少的页面（章节页、金句页、封面页）尤其要注意

### 1.2 全屏与安全区
- 幻灯片尺寸：`100vw × 100vh`
- 顶部 chrome（页码/章节名）和底部 foot（标题/页码）占空间，内容区不要顶到
- padding 参考：桌面端 `5.5vh 5.5vw 9vh`，移动端 `4vh 5vw 10vh`

---

## 二、文字与排版

### 2.1 换行优化
- 所有标题加 `text-wrap: balance`，避免两行时一长一短
- 正文加 `text-wrap: pretty`，避免单个字单独掉到下一行（孤行）
- 卡片标题也用 `text-wrap: balance`

### 2.2 字体层级
- 衬线中文字体（Noto Serif SC）：大标题、金句、数字
- 非衬线中文字体（Noto Sans SC）：正文、说明
- 等宽字体（IBM Plex Mono）：kicker、tag、meta、页码
- 衬线英文字体（Playfair Display）：英文装饰、大数字

### 2.3 标题命名
- 卡片/导航页的标题**不要做目录式罗列**（如「版权 & 数据 & 粉丝 & 风控」）
- 要有概括性，传递价值感（如「AI×IP 的四个关键判断」）
- 格式参考：`第 N 期 AI 调研分享 · 概括性一句话`

---

## 三、组件样式

### 3.1 Tag / Kicker 标签
- 不能太淡看不清，最低要求：
  - 字号：桌面 ≥ 12px，移动端 ≥ 10px
  - 透明度：≥ 0.75（accent 色直接用 1.0）
  - 有边框或背景衬底，不要纯文字飘着
- 推荐样式：细边框 + 大写字母 + 宽字距（letter-spacing: .28em）

### 3.2 Stat 数字统计
- 数字要大、要醒目（衬线字体、粗体）
- 数字上方的标签文字（如「判赔金额」）不能太小太淡：
  - 字号 ≥ 12px
  - 透明度 ≥ 0.7
  - 字重 ≥ 500
- 数字下方的补充说明用更小号、更淡的文字

### 3.3 分隔线 / 小横线
- 40px 那种短装饰横线**没用就删掉**，不要为了加而加
- 真正需要分隔的地方用细的、低透明度的全宽线（opacity: .2）
- 章节之间用大留白 + kicker 标签区分，不靠小横线

### 3.4 卡片
- 历史归档卡片：**最小高度统一**（建议 140px），不能有的高有的矮
- 标题区用 `flex: 1 + align-items: center`，让文字在卡片内垂直居中
- 圆角 16px，内边距 24px
- hover 效果：上移 3px + 边框变色 + 背景略亮

## 四、判例 / 案例页布局

### 4.1 统一格式
所有案例页统一用 **左：案例信息 + 右：裁判要点** 的左右分栏布局：
- 左侧：Case 编号、案例名称、法院/时间、判赔金额、补充说明
- 右侧：2-3 个 callout 卡片，每个卡片一条裁判要点 + 小标题
- 分栏比例：40% / 60%（案例信息少，要点多）

### 4.2 不要做的
- 不要左要点右案例信息（反过来读着别扭）
- 不要案例信息和要点混排
- 不要每个案例页布局都不一样

---

## 五、归属矩阵 / Checklist 页

### 5.1 布局原则
- 左右两列内容多时，**不要用独立卡片上下堆**，会被截断
- 用「表格行」形式：每行一个条目，左侧标签 + 右侧说明
- checklist 用紧凑的行间距，每行一个勾 + 一句话
- 确保内容在一屏内完整显示，或可以流畅滚动

---

## 六、背景装饰（Ghost 数字）

### 6.1 原则
- 大号背景数字是装饰，不能挡正文
- **绝对不能正居中**（刚好挡在文字后面）
- 推荐位置：右下角偏外、左下角偏外、右上角偏外——只露出一部分
- 透明度：0.04 – 0.06（很低，若隐若现）
- 字号可以很大（30-40vw），但只露一角才有张力

---

## 七、滚动条

### 7.1 在哪里滚动
- **移动端：整个幻灯片滚动**（`.slide { overflow-y: auto }`），滚动条在页面最右侧边缘
- **不能让内容区（frame）单独滚动**——滚动条出现在页面中间很丑
- 桌面端：保持设计，不主动加滚动

### 7.2 滚动条样式（必须自定义）
浏览器默认滚动条太粗太丑，必须覆盖：

```css
/* Webkit 浏览器 */
.slide ::-webkit-scrollbar { width: 4px; height: 4px; }
.slide ::-webkit-scrollbar-track { background: transparent; }
.slide ::-webkit-scrollbar-thumb {
  background: rgba(255,255,255,.15);  /* 深色页 */
  border-radius: 4px;
  transition: background .3s ease;
}
.slide ::-webkit-scrollbar-thumb:hover { background: rgba(255,255,255,.28); }
.slide.light ::-webkit-scrollbar-thumb { background: rgba(0,0,0,.15); }  /* 浅色页 */
.slide.light ::-webkit-scrollbar-thumb:hover { background: rgba(0,0,0,.28); }

/* Firefox */
.slide { scrollbar-width: thin; scrollbar-color: rgba(255,255,255,.15) transparent; }
.slide.light { scrollbar-color: rgba(0,0,0,.15) transparent; }
```

要点：
- 4px 超细，几乎隐形
- 轨道完全透明
- 滑块半透明，不抢注意力
- 圆角 4px，圆润
- hover 时加深，有反馈
- 深浅色主题分别适配

## 八、移动端适配

### 8.1 断点
- 平板：900px / 768px
- 手机：600px

### 8.2 移动端必须做的
- 字号整体放大（标题、正文都用 vw 或 max 保底）
- 多列布局变单列（split、grid-3、grid-4 都要塌成 1 列）
- 幻灯片可滚动（`.slide { overflow-y: auto }`），内容多不截断
- frame 不要 overflow:hidden，内容能完整显示
- 内边距适当缩小

### 8.3 移动端不需要的
- 不要改 slide 高度为 auto（会导致横向滑动布局崩）
- 不要让 frame 单独出滚动条

## 九、导航页（index.html）

### 9.1 布局结构
- 顶部：大标题 + 副标题（居中）
- 最新一期：大卡片，突出显示（带 NEW 标记、暖金边框光效）
- 历史归档：**左右双列**
  - 左列：调研分享 · Research（暖金色）
  - 右列：话题分享 · Topic（靛蓝色）

### 9.2 分类规则
- folder 名包含「调研分享」→ 调研分享列
- folder 名包含「话题分享」→ 话题分享列
- 每列内部按日期倒序，最新的带 ★ 标记

### 9.3 数据驱动
- 所有期次数据存在 `weeks.json`，数组第 0 条是最新一期
- `index.html` 读取 `weeks.json` 渲染，同时有 FALLBACK 兜底数据（file:// 打开时用）
- 新增一期：在 `weeks.json` 数组**最前面**加一条，同时更新 FALLBACK

## 十、配色系统

### 10.1 墨水经典（默认深色主题）
- 底色：`#060d1a`（ink-deep）→ `#0a1628`（ink）
- 文字：`#f5f3ef`（paper）/ `#d4cfc5`（soft）/ `#9a958a`（muted）
- 暖金强调：`#c9a96e`（accent）
- 靛蓝强调：`#6b8cff`（indigo，话题分享用）
- 边框线：`rgba(201,169,110,.18)`（暖金边）/ `rgba(255,255,255,.06)`（淡边）

### 10.2 浅色主题（部分页面切换）
- 底色：`#f5f3ef`（paper）
- 文字：`#0a1628`（ink）
- 强调色不变

## 十一、交互动效

### 11.1 翻页
- 横向滑动，整屏切换
- 过渡时间：1.0 – 1.1s，缓出曲线（cubic-bezier(.16,1,.3,1)）
- 操作方式：左右箭头、空格、滚轮、触屏滑动、点击左右边缘

### 11.2 入场动画
- 每页内容依次淡入上移（fadeUp + stagger）
- stagger 间隔：0.1s
- 不要太多元素动，3-5 层足够

### 11.3 演讲者备注
- 每页底部有 `.notes` 区域，存演讲提示
- 默认隐藏，按特定键（如 N）切换显示
- 备注内容要精简：案例名 + 事实 + 一句话核心结论

## 十二、文件结构

```
调研分享/
├── index.html              # 导航页（入口）
├── weeks.json              # 全部期次数据（最前=最新）
├── 更新方法.md             # 怎么加新一期
├── PPT设计规范.md          # 本文档
└── 2026.7.11-调研分享/
    └── ppt/
        └── index.html      # PPT 本体
```

话题分享也放在 `调研分享/` 文件夹下，跟调研分享共用导航页。

## 十三、做新 PPT 自查清单

做完后过一遍，避免反复改：

- [ ] 所有页面内容垂直居中了吗？底部没有大片空白？
- [ ] 标题加了 text-wrap: balance 吗？有没有孤行？
- [ ] tag/kicker/meta 这些小字看得清吗？（字号、透明度够吗）
- [ ] 卡片高度统一吗？
- [ ] 案例页布局统一吗？（左案例 + 右要点）
- [ ] 背景大数字没挡住正文吧？（在角落吗）
- [ ] 移动端内容多的页面能滚动吗？（滚动条在页边，不在内容中间）
- [ ] 滚动条是细的半透明的吗？（不是浏览器默认粗条）
- [ ] 深浅色页面的文字对比度够吗？
- [ ] weeks.json 和导航页 FALLBACK 都更新了吗？
