# 每周更新方法（调研分享 · GitHub Pages 导航站）

## 一句话
每周做三件事：**① 放新一期文件夹 → ② 在 `weeks.json` 最前面加一条 → ③ 提交并推送**。根目录的 `index.html` 导航页不用动（它会自动读 `weeks.json`，把"最新"指向数组第 0 条）。

> ⚠️ **只推日期文件夹、不更新 weeks.json 是不行的**——导航页靠 weeks.json 找最新一期，漏了它，新一期就不会出现在导航里。

## 详细步骤

### 第 1 步：放新一期
把新那期按以下结构放进 `调研分享/` 下：
```
调研分享/
  2026.7.18-调研分享/        ← 文件夹名：YYYY.M.D-调研分享（无前导零、中文"调研分享"）
    ppt/
      index.html              ← 你的网页 PPT
      assets/
        motion.min.js
      （images/ 若为空可不要）
```
> 命名约定：日期用 `YYYY.M.D` 无前导零，后缀固定 `-调研分享`。

### 第 2 步：更新 weeks.json（关键，别漏）
打开仓库根目录的 `weeks.json`，在数组 **最前面** 加一条（不是最后面）：
```json
[
  {
    "folder": "2026.7.18-调研分享",
    "date": "2026-07-18",
    "title": "AI 战略碰撞 · 每周调研分享",
    "ppt": "2026.7.18-调研分享/ppt/"
  },
  {
    "folder": "2026.7.11-调研分享",
    "date": "2026-07-11",
    "title": "AI 战略碰撞 · 每周调研分享（首期）",
    "ppt": "2026.7.11-调研分享/ppt/"
  }
]
```
> 新条目必须放数组【最前】——导航页取 `weeks.json[0]` 作为"最新分享"。

### 第 3 步：提交并推送
因为本 WorkBuddy 环境连不上 GitHub，这步要在**你自己的终端**（能上 GitHub 的电脑）跑：
```bash
cd "E:\Obisidian\obisidian_work\AI分享计划\调研分享"
git add .
git commit -m "add: 2026.7.18 调研分享"
git push
```
> 首次若提示没有 remote，先补一句（把 `<用户名>`/`<TOKEN>` 替换）：
> `git remote add origin https://<TOKEN>@github.com/<用户名>/ai-share-ppt.git`
> 之后每周只需 `git add . && git commit -m "..." && git push`。

### 第 4 步：等待生效
push 后等 1–2 分钟，访问 `https://<用户名>.github.io/ai-share-pkt/` —— 导航页顶部「最新分享」已自动指向新一期，旧的全在「历史归档」。

## 常见疑问
- **要不要改导航页 index.html？** 不用，它动态读 weeks.json，每周只动 weeks.json。
- **只推日期文件夹行不行？** 不行——导航页靠 weeks.json 找最新，漏了它新一期就不会出现在导航里。
- **想改展示标题？** 改对应条目的 `title` 字段即可。
- **本地预览导航页**：直接双击 `调研分享/index.html`（若浏览器拦 fetch，会回退显示首期直链）。
- **临时分享**：之前的 CloudStudio 链接本周这期已能用，适合"先分享、GitHub 后弄"。
